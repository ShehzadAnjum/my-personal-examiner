## Core Expertise

### 1. PhD-Level Explanation Structure

**9-Component Framework** (for Economics 9708):

1. **Definition** (Always visible - critical)
   - Precise technical definition
   - Key variables or components identified
   - Formula if applicable (e.g., PED = % change in Q / % change in P)

2. **Key Terms** (Always visible - vocabulary)
   - 3-5 essential terms with definitions
   - Examples: "elastic demand", "inelastic demand", "unit elastic"

3. **Core Principles** (Always visible - foundational rules)
   - 3-5 fundamental rules or relationships
   - Examples: "If PED > 1, demand is elastic", "Necessities are inelastic"

4. **Real-World Examples** (Collapsible - application)
   - 2-3 concrete, relatable examples
   - Include data/numbers (not just theoretical)
   - **Pattern**: Context ‚Üí Scenario ‚Üí Analysis ‚Üí Conclusion
   - Example: "Insulin (inelastic) vs Designer Handbags (elastic)"

5. **Visual Aids** (Collapsible - spatial understanding)
   - Diagrams: Supply/demand curves, shifts vs movements
   - Graphs: Data visualization (price vs quantity)
   - Tables: Comparison (elastic vs inelastic characteristics)
   - **Rule**: Only include if it clarifies, not decorates

6. **Worked Examples** (Collapsible - step-by-step)
   - 1-2 calculation problems solved step-by-step
   - **Pattern**: Given ‚Üí Find ‚Üí Formula ‚Üí Calculation ‚Üí Answer ‚Üí Interpretation
   - Show all working (don't skip steps)

7. **Common Misconceptions** (Collapsible - error prevention)
   - 3-5 frequent misunderstandings
   - **Pattern**: ‚ùå Misconception ‚Üí ‚úÖ Correction ‚Üí Why it matters
   - Example: "‚ùå Elastic means high demand ‚Üí ‚úÖ Elastic means responsive to price changes"

8. **Practice Problems** (Collapsible - retrieval practice)
   - 3-5 problems increasing in difficulty
   - Include answer outlines (not full solutions - encourage thinking)
   - **Pattern**: Problem ‚Üí Hint (optional) ‚Üí Answer outline

9. **Related Concepts** (Always visible - connections)
   - 3-5 syllabus codes for related topics
   - Explains how concepts connect
   - Examples: PED ‚Üí YED (income elasticity), PED ‚Üí PES (supply elasticity)

### 2. Evidence-Based Teaching Principles

**Principle 1: Spaced Repetition**
- Review saved explanations after 1 day, 3 days, 7 days, 14 days (SM-2 algorithm)
- Don't cram - distributed practice more effective than massed practice

**Principle 2: Retrieval Practice**
- Practice problems force retrieval (more effective than re-reading)
- Answer outlines (not full solutions) encourage thinking
- "Don't just recognize - recall"

**Principle 3: Interleaving**
- Mix different topics in practice (not blocking)
- Example: PED problem ‚Üí YED problem ‚Üí PED problem (interleaved)
- Not: PED ‚Üí PED ‚Üí PED ‚Üí YED ‚Üí YED (blocked)

**Principle 4: Dual Coding**
- Combine words + visuals (more effective than words alone)
- Visual aids must add meaning, not just decoration
- Example: Supply/demand diagram + text explanation

**Principle 5: Elaboration**
- Connect new concepts to prior knowledge
- "Related Concepts" section builds schema
- "Real-World Examples" link abstract to concrete

**Principle 6: Concrete Examples**
- Abstract ‚Üí Concrete progression
- Start with familiar (coffee, insulin) before unfamiliar (elasticity coefficient)

### 3. Practice Problem Design

**Quality Criteria**:
1. **Aligned to Syllabus**: Mirrors Cambridge exam question style
2. **Progressive Difficulty**: Easy ‚Üí Medium ‚Üí Hard
3. **Retrieval-Focused**: Can't be solved by re-reading definition
4. **Real-World Context**: Not just abstract calculations
5. **Feedback-Ready**: Answer outlines explain reasoning

**Example Good Practice Problem**:
```
Problem (Medium Difficulty):
A coffee shop increases price from $3 to $4 per cup. Daily sales
fall from 100 to 80 cups. Calculate PED and advise if the shop
should reverse the price increase.

Answer Outline:
- % change in quantity = (80-100)/100 = -20%
- % change in price = (4-3)/3 = +33.33%
- PED = -20% / +33.33% = -0.6 (inelastic)
- Revenue before: $3 √ó 100 = $300
- Revenue after: $4 √ó 80 = $320
- Advice: Keep price increase (revenue increased by $20)
```

**Why This Is Good**:
- ‚úÖ Requires calculation (not just recognition)
- ‚úÖ Requires interpretation (advise whether to reverse)
- ‚úÖ Real-world context (coffee shop scenario)
- ‚úÖ Answer outline shows reasoning (not just final number)

### 4. Common Misconception Patterns

**Economics 9708 Misconception Types**:

**Type 1: Confusing Terms**
- Example: "Elastic" means "stretchy" (physics) vs "responsive" (economics)
- Correction: Provide economics-specific definition + examples

**Type 2: Causal Direction**
- Example: "High price causes high demand" (confuses supply with demand)
- Correction: Show demand curve (downward sloping - inverse relationship)

**Type 3: Absolute vs Relative**
- Example: "PED is always negative" (sign confusion)
- Correction: Explain why economists use absolute value

**Type 4: Scope Confusion**
- Example: "All necessities have PED < 1" (overgeneralization)
- Correction: Provide counterexamples (organic food - necessity but elastic)

**Pattern for Correction**:
```
‚ùå Misconception: [student's wrong belief]
‚úÖ Correction: [what's actually true]
üí° Why it matters: [consequences of this misunderstanding]
üìö How to remember: [mnemonic or analogy]
```

### 5. Visual Aid Selection Criteria

**Use Diagrams When**:
- ‚úÖ Spatial relationships matter (supply/demand curves, shifts)
- ‚úÖ Process has clear steps (circular flow of income)
- ‚úÖ Comparison of scenarios (elastic vs inelastic on same axes)

**Use Graphs When**:
- ‚úÖ Data visualization needed (UK inflation rate 2010-2025)
- ‚úÖ Trends over time (GDP growth)

**Use Tables When**:
- ‚úÖ Comparing multiple attributes (elastic vs inelastic characteristics)
- ‚úÖ Organizing data (PED values for different goods)

**Don't Use Visuals When**:
- ‚ùå Purely decorative (stock photo of money)
- ‚ùå More confusing than text (overly complex flowchart)
- ‚ùå Redundant with text (table that just repeats paragraph)

### 6. Content Scaffolding (Simple ‚Üí Complex)

**Progression Levels**:

**Level 1: Definition** (What is it?)
- Technical definition
- Key components
- Formula if applicable

**Level 2: Examples** (What does it look like?)
- Concrete, familiar examples
- Real-world scenarios with data

**Level 3: Application** (How do I use it?)
- Worked examples with step-by-step
- Practice problems (guided)

**Level 4: Analysis** (Why does this matter?)
- Common misconceptions addressed
- Connections to other concepts
- Real-world implications

**Level 5: Evaluation** (When should I use this vs that?)
- Comparison with related concepts (PED vs YED)
- Limitations of the concept
- Advanced practice problems

**Rule**: Don't skip levels. Student must understand Level N before Level N+1.
